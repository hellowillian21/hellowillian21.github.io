Babel转码器笔记

Babel (babeljs.io) 是一个广为使用的es6转码器,可以将es6代码转为es5代码，使得编写es6程序不用担心现有环境是否支持。以下为学习阮一峰babel摘要笔记。

1、配置文件 .babelrc

Babel的配置文件是.babelrc。该文件基本格式:

{
    "presets": [],
    "plugins": []
}

presets字段设定转码规则， 官方提供以下的规则集，可以根据需要进行安装。

# 最新转码规则
npm install babel-preset-latest --save-dev

# react 转码规则
npm install babel-preset-react --save-dev

# 不同阶段语法提案的转码规则（共有四个阶段），选装一个
npm install babel-preset-stage-0

npm install babel-preset-stage-1

npm install babel-preset-stage-2

npm install babel-preset-stage-3

然后将这些规则加入.babelrc中。

{
"presets": [
    "latest",
    "react",
    "stage-2"
],
"plugins": []
}

2、命令行转码babel-cli

Babel提供babel-cli工具，用于命令行转码。在项目中安装babel-cli

npm install babel-cli --save-dev

然后改写package.json

{
  "name": "test",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "build": "babel src -d lib"
  },
  "author": "hellowillian21",
  "license": "ISC",
  "devDependencies": {
    "babel-cli": "^6.26.0",
    "babel-preset-latest": "^6.24.1",
    "babel-preset-react": "^6.24.1",
    "babel-preset-stage-3": "^6.24.1"
  }
}

执行转码命令

npm run build

那么babel就会把src下的js文件转码后存放在我们指定的lib文件夹下。

3、babel-polyfill

由于Babel只转换新的语法而不转换新的API,所以需要使用babel-polyfill为当前环境提供一个垫片

安装命令如下:

npm install babel-polyfill --save

然后在脚本头部加入如下代码:

import 'babel-polyfill'
//或者
require('babel-polyfill')
